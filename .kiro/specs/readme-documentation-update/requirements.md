# 需求文檔：README 文檔更新

## 簡介

此功能旨在更新 LINE Desktop NVDA 附加元件的四個語言版本的 README 文檔（英文、日文、泰文、繁體中文），以反映新增的快速鍵功能、虛擬視窗說明，並改善文檔結構。

## 術語表

- **README**: 附加元件的使用說明文檔
- **快速鍵 (Keyboard Shortcut)**: 用於執行特定操作的鍵盤組合鍵
- **虛擬視窗 (Virtual Window)**: 附加元件提供的模擬視窗介面，用於改善無障礙體驗
- **輸入手勢 (Input Gestures)**: NVDA 中用於自訂快速鍵的設定介面
- **OCR**: 光學字元辨識技術

## 需求

### 需求 1：新增檔案儲存快速鍵說明（訊息操作）

**使用者故事：** 作為使用者，我想要了解如何使用快速鍵開啟另存新檔對話框來儲存檔案訊息，以便快速儲存收到的檔案。

#### 驗收標準

1. THE README 文檔 SHALL 在「訊息操作」快速鍵表格中新增 `NVDA+Windows+K` 快速鍵說明
2. THE 快速鍵說明 SHALL 標示類別為「附加元件」
3. THE 快速鍵說明 SHALL 包含操作描述「開啟另存新檔對話框」或「儲存檔案」
4. THE 快速鍵說明 SHALL 包含額外說明，提醒使用者檔案上傳後只能下載 7 天

### 需求 1.5：新增儲存語音訊息快速鍵說明（媒體播放器）

**使用者故事：** 作為使用者，我想要了解如何使用快速鍵儲存語音訊息，以便快速儲存收到的語音訊息。

#### 驗收標準

1. THE README 文檔 SHALL 在「媒體播放器」快速鍵表格中新增 `NVDA+Windows+K` 快速鍵說明
2. THE 快速鍵說明 SHALL 標示類別為「附加元件」
3. THE 快速鍵說明 SHALL 包含操作描述「儲存語音訊息」或「另存新檔」

### 需求 2：新增播放語音訊息快速鍵說明

**使用者故事：** 作為使用者，我想要了解如何使用快速鍵播放語音訊息，以便快速播放收到的語音訊息。

#### 驗收標準

1. THE README 文檔 SHALL 在快速鍵表格中新增 `NVDA+Windows+P` 快速鍵說明
2. THE 快速鍵說明 SHALL 標示類別為「附加元件」
3. THE 快速鍵說明 SHALL 包含操作描述「播放語音訊息」

### 需求 3：新增輸入手勢調整說明

**使用者故事：** 作為使用者，我想要知道如何自訂快速鍵，以便根據個人習慣調整快速鍵配置。

#### 驗收標準

1. THE README 文檔 SHALL 在快速鍵章節開頭或適當位置新增說明文字
2. THE 說明文字 SHALL 告知使用者可透過「NVDA 功能表 → 偏好 → 輸入手勢」調整快速鍵
3. THE 說明文字 SHALL 以適合該語言的方式呈現（英文、日文、泰文、繁體中文）

### 需求 4：新增虛擬視窗說明章節

**使用者故事：** 作為使用者，我想要了解虛擬視窗的功能和使用方式，以便更有效地使用附加元件提供的無障礙功能。

#### 驗收標準

1. THE README 文檔 SHALL 新增一個獨立章節說明虛擬視窗功能
2. THE 虛擬視窗章節 SHALL 說明虛擬視窗的用途和運作方式
3. THE 虛擬視窗章節 SHALL 提供使用虛擬視窗的操作指引
4. THE 虛擬視窗章節 SHALL 放置在適當的文檔位置（建議在「使用技巧與提示」之後）

### 需求 5：重組音效提示內容

**使用者故事：** 作為使用者，我想要在使用技巧章節中看到音效提示說明，以便在閱讀使用指引時一併了解音效提示的意義。

#### 驗收標準

1. THE README 文檔 SHALL 將現有「音效提示」章節的內容移至「使用技巧與提示」章節
2. THE 音效提示內容 SHALL 保持完整，包含所有原有說明
3. THE 音效提示內容 SHALL 整合為「使用技巧與提示」章節的子內容
4. THE README 文檔 SHALL 移除獨立的「音效提示」章節標題

### 需求 6：多語言一致性

**使用者故事：** 作為多語言使用者，我想要各語言版本的文檔結構和內容保持一致，以便無論使用哪種語言都能獲得相同的資訊。

#### 驗收標準

1. THE 文檔更新 SHALL 同時套用至英文、日文、泰文、繁體中文四個版本
2. THE 各語言版本 SHALL 保持相同的章節結構和內容順序
3. THE 各語言版本 SHALL 使用該語言的慣用表達方式
4. THE 快速鍵組合 SHALL 在所有語言版本中保持一致（如 `NVDA+Windows+K`）

### 需求 7：文檔格式一致性

**使用者故事：** 作為使用者，我想要文檔格式保持一致，以便更容易閱讀和理解文檔內容。

#### 驗收標準

1. THE 新增內容 SHALL 遵循現有文檔的 Markdown 格式規範
2. THE 快速鍵表格 SHALL 使用與現有表格相同的格式
3. THE 章節標題 SHALL 使用與現有章節相同的層級結構
4. THE 特殊標記（如 `> [!NOTE]`, `> [!WARNING]`）SHALL 在適當位置使用
